--- CUSTOM THEME ---

Design a futuristic, modern dark-first theme inspired by crisis/emergency monitoring dashboards.
The aesthetic should feel like a high-tech command center — clean, data-rich, and authoritative.

Color palette (app_colors.dart):
  - Background primary: deep navy/charcoal (#0A0E1A or similar very dark blue-black)
  - Background secondary: slightly lighter card surface (#111827)
  - Background tertiary: elevated surface (#1E293B)
  - Accent primary: vivid cyan/electric blue (#00D4FF) — used for active states, CTAs, map markers
  - Accent secondary: soft teal (#2DD4BF) — used for success, verified badges
  - Danger/critical: vivid red-orange (#FF3B5C) — used for high severity, SOS
  - Warning: amber (#FFAA00) — used for medium severity alerts
  - Success: emerald green (#10B981)
  - Text primary: white with 95% opacity
  - Text secondary: white with 60% opacity
  - Text tertiary: white with 40% opacity
  - Divider: white with 8% opacity
  - Severity gradient: green (1-2) → yellow (3-4) → orange (5-6) → red (7-8) → deep red (9-10)

Light theme: provide a clean light variant using the same accent colors but with white/gray backgrounds.

Typography (app_typography.dart):
  - Use "Inter" font family (import from Google Fonts or bundle)
  - Headlines: semibold/bold, generous letter-spacing
  - Body: regular weight, comfortable line-height (1.5)
  - Labels/captions: medium weight, slightly condensed
  - Monospace variant for data readouts (timestamps, coordinates): "JetBrains Mono" or "Fira Code"

Spacing (app_spacing.dart):
  - Use an 4px base grid: xs=4, sm=8, md=16, lg=24, xl=32, xxl=48
  - Border radius: sm=8, md=12, lg=16, xl=24 (generous rounding, modern feel)

ThemeData (app_theme.dart):
  - Build complete ThemeData for both light and dark modes
  - Style ElevatedButton: rounded (borderRadius 12), height 52, no elevation, gradient option
  - Style Card: rounded (borderRadius 16), subtle border (1px white/8%), no default elevation
  - Style AppBar: transparent/blurred background, no elevation
  - Style BottomNavigationBar: transparent background, rounded indicator
  - Style TextField/InputDecoration: filled, rounded (borderRadius 12), subtle border on focus
  - Style BottomSheet: rounded top corners (borderRadius 24)
  - Smooth page transitions (fade + slide)

--- CORE NETWORKING (api_client.dart) ---

Build a Dio-based API client:
  - Base URL loaded from flutter_dotenv (.env file with API_BASE_URL)
  - Request interceptor: attach JWT from secure storage as "Authorization: Bearer <token>"
  - Response interceptor: handle 401 (clear token, redirect to login)
  - Error interceptor: map DioExceptions to typed ApiException subclasses
  - Timeout: connect=10s, receive=15s
  - Expose as a Riverpod provider

API endpoints constants (api_endpoints.dart) — define ALL backend paths:

--- AUTH FEATURE (COMPLETE) ---

Implement the full Google Sign-In flow:
1. LoginScreen: beautiful full-screen with app logo, tagline ("Real-time crisis awareness"),
   subtle animated background (gradient shift or particle effect using flutter_animate),
   and a prominent "Continue with Google" button styled with the custom theme.
2. On tap: firebase_auth + google_sign_in → get Firebase ID token.
3. Send POST /api/auth/google with { "id_token": "<firebase_token>" }.
4. Receive { "token": "<jwt>", "user": { ... } }.
5. Store JWT in secure storage.
6. Store user in a Riverpod StateNotifier/AsyncNotifier (auth_controller.dart).
7. Navigate to /home.

Auth controller (Riverpod AsyncNotifier):
  - state: AsyncValue<User?>
  - signInWithGoogle() → calls repository → updates state
  - signOut() → clears storage, resets state, navigates to /login
  - checkAuthStatus() → reads stored JWT, fetches user from GET /api/users

Auth repository (abstract + impl):
  - googleSignIn(String idToken) → AuthResponse
  - getUser() → User
  - updateUser(String name, List<SavedLocation> locations) → User
  - signOut() → void
